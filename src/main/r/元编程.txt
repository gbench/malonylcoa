# ‌元编程（Metaprogramming）‌是一种编程范式，允许程序在运行时生成或修改代码。
# 元编程通过操作程序实体，在编译时生成运行时所需的常数、类型和代码。它通过利用编程语言提供的模板机制，
# 在编译时生成程序，然后再进行编译，最终生成目标代码。‌

# 加载库
library(data.table)

# 元编程的特点就是它可以通过编写不符合标注R标准语法结构，通常是一系列更为精简的DSL式样的代码模板片段，而去进行的编程设计。
# 比如：rbindlist()， lapply(FUN=read.csv) ，head(n=3)，。。。 代码片段（R的叫法为call对象），注意：他们都缺少目标操作对象参数ｘ
＃元编程的目的与技巧，就是将ｘ给填写片段然后组织生成合法的R语言程序，技巧即使以什么方式去填写，如何通过程序的编码手段去动态组合
＃这些片段而进行动态的。此处的简单示例将以 Reduce 规约的形式去量变地组合引起质变的形式来复合出新的代码，即所以规约的形式去构造
＃源码层级上的演绎效果的元编程模式。让程序组织代码片段（代码功能模块）以指挥源码的方式进程程序设计。
# 
# 生成函数复合表达式,逆向管道，之所以使用逆向管道，是为了让打印出的生成的中间代码数学传统的f.g.h的顺序的思维习惯相一致
expression( rbindlist(), lapply(FUN=read.csv), head(n=3), sort(decreasing=T), grep(pattern="\\.csv$", value=T) ) |> # 管道代码
  Reduce(\(f, x, keys=names(formals(get(f[[1]]))), k=setdiff(keys, names(f[-1]))[1] ) {
       f[[k]]<-x; print(f); f # 编辑表达式;生成程序代码;打印代码表达式&返回生成的表达式
  } , x=_, init=quote(list.files()), right=T) |> # 生成表达式语句
eval() # 运行程序生成的代码
